# Omi整理係

毎朝7:30 JSTに実行されるリモートトリガー。Omi APIから昨日の会話データを取得し、カテゴリ別に整理してこのリポジトリにpushする。

## データ取得

- Omi API Key: `omi_mcp_7b1bbb69419a6934e18056270f7ed6c6`
- Base URL: `https://api.omi.me/v1/mcp`
- 会話一覧: `GET /conversations?start_date=YYYY-MM-DD&end_date=YYYY-MM-DD&limit=50`
  - end_dateは排他的。JSTの1日分を取るには前日UTC〜翌日UTCでカバーし、JSTで絞る
- 会話詳細: `GET /conversations/{id}`（full transcript付き）

## 日付とタイムゾーン

- APIのstart_date/end_dateはUTC基準
- 昨日のJSTデータを取得するには: `start_date=(JST昨日-1日のUTC日付)&end_date=(JST今日のUTC日付+1)` で広めに取り、started_atをJST変換して昨日分のみ抽出する
- 簡易計算: JSTの日付Dの会話は、UTC `D-1` 15:00 〜 `D` 14:59 の範囲

## カテゴリ分類

Omi APIのcategoryフィールドを参考にしつつ、以下の6カテゴリに再分類する:

| カテゴリ | 抽出内容 |
|---------|---------|
| 食事 | 何を食べた・飲んだか、推定カロリー、時間帯 |
| 運動 | 何をしたか、推定消費カロリー |
| 健康 | 体調、通院、睡眠の話 |
| 仕事 | 打ち合わせ、タスク、決定事項 |
| 家族 | 快斗の出来事、妻との会話のポイント |
| その他 | 上記に当てはまらないもの |

## 食事カテゴリの特別処理

食事に関する情報はoverviewだけでは不十分な場合がある（「くら寿司行った」程度の記述）。
食事の記述がありそうな会話は、full transcriptを取得して具体的なメニューを抽出する。

## 誤字修正

Omiの音声認識は誤字が多い。特に以下は修正する:
- 海斗 → 快斗（子どもの名前）
- その他、文脈から明らかな誤変換は修正
